Useful Git Commands

  1. Create a branch, we do not work on the main branch. You can create it from the github branch dropdown or use a git command 'git checkout -b my-branch'. Do a "git checkout -help" to know more.
    $ git checkout -b my-branch
  2. Adding the upstream for the repository 'rk_web_fs' :
    $ git remote add upstream https://github.com/katukojwalark/rk_web_fs
  3. Adding the Origin:
    $ git remote add origin https://github.com/katukojwalark/rk_web_fs.git (copy this Link from the drop down code from your main branch)
  4. Check for the Upstream and Origin:
    $ git remote -v
  5. Fetch upstream: downloads commits, files, and refs from a remote repository into your local repo. Fetching is what you do when you want to see what everybody else has been working on.
    $ git fetch upstream
  6. Update your branch from the upstream:
    $ git rebase upstream/main
  7. Make your code changes. You have already created the branch. If not created, using '-b' will create and switchest to it.
    $ git checkout -b my-branch-name
  8. You have made your code changes check the files that are modified or added using,
    $ git status
  9. Now you can stage the files you are about to commit using $ git add file-name or you can stage all,
    $ git add . (or) $ git add file-name
  10. You have staged your changes and are ready to commit to a branch. Double check the files that you want to commit using the,
    $ git status
  11. Commit the changes to the branch.
    $ git commit or git commit -m "my message describing the commiting changes"
  12. You have commited the changes to a branch. You need to share it to others to get the code reviews done.
    $ git push origin rk-branch
  13. Go to GitHub and raise PR from there. GitHub highlights a message like " rk-branch had recent pushes less than a minute ago" with a button "Compare & pull request" click it to raise a PR.
  14. On successful review, merge your changes to the main branch.
*If you are the owner of the repository, you can directly submit the changes to the 'main' branch. When you submit to a main branch no PR is created.*
